Strings are IMMUTABLE — every method that 'changes' a string returns a NEW string. The original variable is unchanged unless you reassign it. ★ marks newly added methods.

ArrayList is MUTABLE. Methods like add(), remove(), set() modify the list in-place. List.of() creates IMMUTABLE lists. subList() returns a VIEW, not a copy.

8 patterns

Key gotcha: Arrays.asList() returns a FIXED-SIZE list — you cannot add/remove elements. Wrap in new ArrayList<>() to make it fully mutable.

ConversionSyntaxDescriptionImportant Notes
Array → List (fixed)Arrays.asList(arr)Fixed-size List backed by array; no add/remove.WARN add() and remove() throw UnsupportedOperationException. set() works. Changes to list affect original array.

contains()queue.contains('x')Returns true if element exists in queue.WARN O(n) linear scan.
